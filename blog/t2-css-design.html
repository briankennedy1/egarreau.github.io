Home is Where the Positioning Is

Fixed, Relative, and Absolute Positioning in CSS

March 29th, 2015

Every element of an HTML document has a home. For most elements, their home is static. The default positioning CSS for all elements is:

position:static;

All this means is that the element lives on its own, in the same neighborhood as all the other elements, happy in its home.

But some elements get antsy (you know how elements can be). An element that wants to leave its current home, but doesn't want any other elements moving in on its space, gets the CSS

position:relative;

This allows the element to be moved around, but all the other elements will respect its home and not try to take over the space that it originally occupied.

More adventurous elements can be assigned the position fixed, which takes it out completely from the flow of the the other elements and gives it a new home on your browser window. In this context we can think of the browser window and the webpage you're viewing as two sheets of paper on top of one another (yeah, I'm mixing metaphors. I like to live dangerously). The top sheet (the browser) is transparent and usually doesn't matter. But when you use position:fixed; the element is removed from the bottom sheet and put on the transparent top sheet. The other elements act like it doesn't exist, and the fixed element floats over them like a happy cloud (keep that metaphor mixer churnin'). If you ever see a navigation bar that stays at the top of the page while you scroll down, it's likely that's a fixed element.

Then again, some elements never quite make it out of their parents' house. This kind of elements gets the position absolute, which allows it to move around in relation to its closest ancestor element that has a relationship other than static.
